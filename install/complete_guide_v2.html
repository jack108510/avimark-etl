1. Overview & Requirements

This tool reads Avimark Classic's proprietary V2$ binary data files and uploads them to a Supabase PostgreSQL database. It syncs 30 tables totaling ~2.58 million rows nightly at 2:00 AM via Windows Task Scheduler.

Requirements:

  • Windows 7/10/11 or Server 2012+
  • Node.js v18+ LTS — https://nodejs.org
  • Avimark Classic installed (V2$ files accessible on disk)
  • Internet connection (for Supabase uploads)
  • Supabase project URL + service key

Key Info:

  • GitHub: https://github.com/jack108510/avimark-etl
  • Supabase: https://rnqhhzatlxmyvccdvqkr.supabase.co
  • Dashboard: https://jack108510.github.io/avimark-etl/
  • Nightly Sync: 2:00 AM via Windows Task Scheduler
  • Sync Log: C:\avimark-etl\sync.log

2. Step-by-Step Installation

1 Install Node.js
Download LTS from https://nodejs.org. Run installer with defaults. Verify: node --version
2 Clone or copy the project folder to the target machine 
git clone https://github.com/jack108510/avimark-etl.git C:\avimark-etl
-- or --
xcopy E:\avimark-etl C:\avimark-etl /E /I
3 Install dependencies 
cd C:\avimark-etl
npm install --production
4 Configure .env
Create C:\avimark-etl\.env in Notepad: 
SUPABASE_URL=https://rnqhhzatlxmyvccdvqkr.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGci...  (your service key from Supabase dashboard → Settings → API)
AVIMARK_DATA_DIR=C:\AVImark       (path to V2$ files on THIS machine)
⚠ Security: Never commit .env to git or share the service key publicly.
5 Create Supabase tables
Run the SQL from Section 4 in the Supabase SQL Editor (Dashboard → SQL Editor → New query).
6 Test the ETL 
cd C:\avimark-etl
node src/etl.js --all
Expected: ~2.58M rows across 30 tables, zero errors. Takes ~15 minutes on first run.
7 Create the nightly scheduled task (see Section 3)
Done! The ETL syncs every night at 2:00 AM. Check logs: C:\avimark-etl\sync.log

3. Windows Scheduled Task Setup

sync.bat is included in the project root. It logs timestamps and runs the ETL:

@echo off
echo [%date% %time%] Starting Avimark ETL sync... >> C:\avimark-etl\sync.log
cd /d C:\avimark-etl
node src/etl.js --all >> sync.log 2>&1
echo [%date% %time%] Sync complete (exit code: %ERRORLEVEL%) >> sync.log

Open Command Prompt as Administrator and run:

schtasks /create /tn "Avimark ETL Nightly Sync" /tr "C:\avimark-etl\sync.bat" /sc daily /st 02:00 /f

Useful commands:

schtasks /query /tn "Avimark ETL Nightly Sync"    -- Check status
schtasks /run /tn "Avimark ETL Nightly Sync"       -- Run now (test)
schtasks /change /tn "Avimark ETL Nightly Sync" /st 03:00  -- Change time
schtasks /delete /tn "Avimark ETL Nightly Sync" /f         -- Remove

Technical Notes

  • SERVICE.V2$ — @21 has TDateTime (reliable post-record ~370K / post-2019; earlier records stamped 2019-02-03)
  • PRICE.V2$ — @21 has last-modified TDateTime per price record (reliable across full file)
  • PO.V2$ — Alternating header/line-item pairs. Costs at @32 as Delphi currency (/10000)
  • VISIT.V2$ — @21 = visit date, @45 = doctor initials (Pascal string). @40 is NOT a range pointer
  • All V2$ files — Fixed-record Delphi flat files with proprietary encoding

6. Manual Commands & Troubleshooting

Manual sync commands

cd C:\avimark-etl
node src/etl.js --all                  # Sync all 30 tables
node src/etl.js --table clients        # Sync one table
node src/etl.js --table prices --limit 100  # Sync 100 prices only

Check sync log

type C:\avimark-etl\sync.log                      # Command Prompt
Get-Content C:\avimark-etl\sync.log -Tail 20       # PowerShell (last 20 lines)

Common issues

ErrorFix
ENOENT: File not foundUpdate AVIMARK_DATA_DIR in .env to the path containing V2$ files
Missing SUPABASE_URLCheck .env has both SUPABASE_URL and SUPABASE_SERVICE_KEY
node is not recognizedInstall Node.js from nodejs.org and restart your terminal
Scheduled task doesn't runOpen taskschd.msc, check task is enabled and user has permissions
Table doesn't exist errorRun the SQL from Section 4 in Supabase SQL Editor
Sync takes too longFirst run is ~15 min. Subsequent runs do full replace. Consider syncing subsets.
