1. Overview & Requirements

This tool reads Avimark Classic's proprietary V2$ binary data files and uploads them to a Supabase PostgreSQL database. It syncs 29 tables totaling ~2.55 million rows nightly.

Requirements:

  • Windows 7/10/11 or Server 2012+
  • Node.js v18+ LTS — https://nodejs.org
  • Avimark Classic installed (V2$ files accessible on disk)
  • Internet connection (for Supabase uploads)
  • Supabase project URL + service key

2. Step-by-Step Installation

1 Install Node.js
Download LTS from https://nodejs.org. Run installer with defaults. Verify: node --version
2 Copy the project folder to the target machine 
xcopy E:\avimark-etl C:\avimark-etl /E /I
3 Install dependencies 
cd C:\avimark-etl
npm install --production
4 Configure .env
Open C:\avimark-etl\.env in Notepad: 
SUPABASE_URL=https://rnqhhzatlxmyvccdvqkr.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGci...  (your service key from Supabase dashboard)
AVIMARK_DATA_DIR=C:\AVImark       (path to V2$ files on THIS machine)
5 Create Supabase tables
Run the SQL from Section 4 in the Supabase SQL Editor.
6 Test 
cd C:\avimark-etl
node src/etl.js --dry-run --all
If that works, run the real sync: 
node src/etl.js --all
7 Create the nightly scheduled task (see Section 3)
Done! The ETL syncs every night at 2:00 AM. Logs: C:\avimark-etl\sync.log

3. Windows Scheduled Task Setup

Open Command Prompt as Administrator and run:

schtasks /create /tn "Avimark ETL Nightly Sync" /tr "C:\avimark-etl\sync.bat" /sc daily /st 02:00 /f

To verify it was created:

schtasks /query /tn "Avimark ETL Nightly Sync"

To run it manually (test):

schtasks /run /tn "Avimark ETL Nightly Sync"

To delete it:

schtasks /delete /tn "Avimark ETL Nightly Sync" /f

To change the time (e.g. 3:00 AM):

schtasks /change /tn "Avimark ETL Nightly Sync" /st 03:00

6. Manual Commands & Troubleshooting

Manual sync commands

cd C:\avimark-etl
node src/etl.js --all                  # Sync all 29 tables
node src/etl.js --table clients        # Sync one table
node src/etl.js --dry-run --all        # Parse without uploading
node src/etl.js --table prices --limit 100  # Sync 100 prices only

Check sync log

type C:\avimark-etl\sync.log           # Command Prompt
Get-Content C:\avimark-etl\sync.log -Tail 20  # PowerShell

Common issues

ErrorFix
ENOENT: File not foundUpdate AVIMARK_DATA_DIR in .env
Missing SUPABASE_URLCheck .env has both URL and service key
node is not recognizedInstall Node.js from nodejs.org
Scheduled task doesn't runOpen taskschd.msc, check task is enabled and user has permissions
Table doesn't exist errorRun the SQL from Section 4 in Supabase SQL Editor
