Avimark ETL — Installation Guide

Automated nightly sync: Avimark V2$ flat files → Supabase SQL database

Overview

This tool reads Avimark Classic's proprietary V2$ binary data files and uploads them to a Supabase PostgreSQL database every night. It parses 29 tables covering clients, patients, services, billing, medical records, prescriptions, vaccines, and more — totaling ~2.5 million rows.

Requirements

✅ Windows 7/10/11 or Server 2012+ ✅ Node.js v18+ (LTS) ✅ Avimark Classic installed (V2$ files accessible) ✅ Internet connection (for Supabase uploads) ✅ Supabase project with tables created

Installation Steps

1 Install Node.js
Download from https://nodejs.org — choose the LTS version. Run the installer with default settings. Verify: open Command Prompt and run node --version
2 Copy the project folder
Copy the entire avimark-etl folder to the target machine. Recommended location: C:\avimark-etl 
:: From a USB drive:
xcopy E:\avimark-etl C:\avimark-etl /E /I

:: Or over the network:
xcopy \\SERVER\share\avimark-etl C:\avimark-etl /E /I
3 Configure the .env file
Open C:\avimark-etl\.env in Notepad and verify these values: 
SUPABASE_URL=https://rnqhhzatlxmyvccdvqkr.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGci... (your service key)
AVIMARK_DATA_DIR=C:\AVImark     (path to V2$ files on THIS machine)
⚠️ Important: The AVIMARK_DATA_DIR must point to where the V2$ files are on the new machine. If Avimark is installed on a different drive or path, update accordingly.
4 Run the installer
Double-click install\install.bat or run from Command Prompt: 
cd C:\avimark-etl
install\install.bat
The installer will:
  • Verify Node.js is installed
  • Install npm dependencies
  • Detect the AVImark data folder
  • Run a test parse to verify everything works
  • Create a Windows Scheduled Task for nightly sync at 2:00 AM
5 Run the first full sync
cd C:\avimark-etl
node src/etl.js --all
This takes ~15 minutes to push all 2.5M rows. Subsequent nightly runs take the same time (full upsert).
✅ Done! The ETL will now sync every night at 2:00 AM automatically.

Troubleshooting

Sync fails with "ENOENT" or "File not found"

The AVIMARK_DATA_DIR in .env is wrong. Update it to the correct path where V2$ files are stored.

Sync fails with "Missing SUPABASE_URL"

The .env file is missing or incomplete. Make sure both SUPABASE_URL and SUPABASE_SERVICE_KEY are set.

Scheduled task doesn't run

Open Task Scheduler (taskschd.msc), find "Avimark ETL Nightly Sync", and check:

  • The task is enabled
  • "Run whether user is logged on or not" is selected
  • The user account has permission to run

Checking the sync log

type C:\avimark-etl\sync.log
:: Or open in Notepad:
notepad C:\avimark-etl\sync.log

Running a manual sync

cd C:\avimark-etl
node src/etl.js --all              :: Sync everything
node src/etl.js --table clients    :: Sync one table
node src/etl.js --dry-run --all    :: Parse without uploading
